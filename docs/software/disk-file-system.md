# 磁盘与文件系统

磁盘管理和文件系统是 Windows 使用的基础知识。分区、格式化、文件类型，这些操作用对了能让数据更安全，用错了可能丢数据。

## 文件系统

### 快速带你了解常用的文件系统

NTFS、FAT32、exFAT、ReFS，Windows 支持好几种文件系统，每种都有不同的特点和适用场景：

- **NTFS** - Windows 默认文件系统，支持权限控制、加密、压缩，日常使用首选
- **FAT32** - 兼容性最好，但单文件不能超过 4GB，适合 U 盘
- **exFAT** - 跨平台兼容，没有 4GB 限制，U 盘和移动硬盘推荐
- **ReFS** - 新一代文件系统，数据完整性更强，适合服务器

### 文件系统详细对比

#### NTFS（New Technology File System）

NTFS 是 Windows 的默认文件系统，从 Windows NT 3.1 开始使用，至今仍是主流。

**技术特性：**
- **最大卷大小**：256TB（理论值），实际受分区表限制
- **最大文件大小**：256TB
- **文件名长度**：255 个字符
- **日志功能**：支持，断电后可恢复文件系统结构
- **权限控制**：支持 ACL（访问控制列表），可精确控制每个用户/组的权限
- **加密**：支持 EFS（加密文件系统）
- **压缩**：支持文件和文件夹级别的压缩
- **硬链接**：支持，多个文件名指向同一份数据
- **符号链接**：支持，类似快捷方式但更底层
- **磁盘配额**：支持，可限制用户使用的磁盘空间
- **卷影副本**：支持，可恢复到之前的版本

**适用场景：**
- Windows 系统盘（必须用 NTFS）
- 内部硬盘和 SSD
- 需要权限控制的共享文件夹
- 需要加密的敏感数据

**注意事项：**
- macOS 只能读取 NTFS，写入需要第三方工具（如 Paragon NTFS、Mounty）
- Linux 可以读写 NTFS，但性能不如原生文件系统
- 不适合 U 盘（频繁读写会缩短寿命，因为日志功能会增加写入量）

#### FAT32（File Allocation Table 32）

FAT32 是最古老的文件系统之一，1996 年随 Windows 95 OSR2 发布。虽然老，但兼容性无敌。

**技术特性：**
- **最大卷大小**：8TB（Windows 格式化限制为 32GB）
- **最大文件大小**：4GB（这是最大限制）
- **文件名长度**：8.3 格式（8 个字符文件名 + 3 个字符扩展名），支持长文件名但有兼容性问题
- **日志功能**：不支持，断电可能导致数据丢失
- **权限控制**：不支持
- **加密**：不支持
- **压缩**：不支持

**适用场景：**
- U 盘（如果不需要存大于 4GB 的文件）
- SD 卡（相机、行车记录仪等设备）
- 需要最大兼容性的场景（老设备、游戏机、车载系统）
- EFI 系统分区（通常是 FAT32 格式）

**注意事项：**
- 单文件不能超过 4GB，这是最大限制
- 不适合大容量硬盘（格式化速度慢，性能差）
- 没有日志功能，断电后文件系统容易损坏

#### exFAT（Extended File Allocation Table）

exFAT 是微软 2006 年推出的文件系统，专门为闪存设备设计。解决了 FAT32 的 4GB 限制，同时保持了较好的兼容性。

**技术特性：**
- **最大卷大小**：128PB（理论值）
- **最大文件大小**：16EB（理论值），实际无限制
- **文件名长度**：255 个字符
- **日志功能**：不支持（但有原子操作保护）
- **权限控制**：不支持
- **加密**：不支持
- **压缩**：不支持
- **簇大小**：支持从 512 字节到 32MB

**适用场景：**
- 大容量 U 盘（需要存大于 4GB 的文件）
- 移动硬盘（跨平台使用）
- SD 卡（大于 32GB 的 SD 卡推荐 exFAT）
- macOS 和 Windows 之间传文件

**注意事项：**
- Windows XP 需要安装补丁才能支持
- 部分老设备可能不支持 exFAT
- 没有日志功能，安全性不如 NTFS
- 断电后数据恢复难度较大

#### ReFS（Resilient File System）

ReFS 是微软 2012 年推出的新一代文件系统，主要面向服务器和企业用户。

**技术特性：**
- **最大卷大小**：35PB
- **最大文件大小**：16EB
- **数据完整性**：使用校验和检测并修复数据损坏
- **自动修复**：配合 Storage Spaces 可自动修复损坏的数据
- **去重**：支持块级去重，节省存储空间
- **弹性**：对元数据损坏有更强的恢复能力

**适用场景：**
- Windows Server（推荐用于文件服务器）
- Hyper-V 虚拟机存储
- 大数据存储场景
- 需要高数据完整性的场景

**注意事项：**
- Windows 家庭版不支持 ReFS
- 不能作为系统盘（不能启动 Windows）
- 与部分旧软件不兼容

### 文件系统选择建议

| 使用场景 | 推荐文件系统 | 原因 |
|---------|-------------|------|
| Windows 系统盘 | NTFS | 必须使用，支持所有 Windows 功能 |
| 内部数据盘 | NTFS | 权限控制、日志、加密 |
| U 盘（小于 32GB） | FAT32 | 兼容性最好 |
| U 盘（大于 32GB） | exFAT | 没有 4GB 限制 |
| 移动硬盘（跨平台） | exFAT | Windows/macOS 都能读写 |
| Mac 和 Windows 共享 | exFAT | 双平台兼容 |
| 相机 SD 卡 | exFAT（大于 32GB）/ FAT32（小于 32GB） | 设备兼容性 |
| Linux 系统 | ext4 | Linux 原生文件系统 |
| Windows Server | NTFS 或 ReFS | 数据完整性 |

### 文件系统格式化方法

**方法一：文件资源管理器**
1. 打开"此电脑"
2. 右键点击要格式化的磁盘
3. 选择"格式化"
4. 选择文件系统（NTFS/exFAT/FAT32）
5. 分配单元大小保持默认
6. 点击"开始"

**方法二：磁盘管理**
1. 右键"开始"按钮 → "磁盘管理"
2. 右键点击分区
3. 选择"格式化"
4. 选择文件系统和卷标
5. 确认

**方法三：命令行（diskpart）**
```cmd
# 打开命令提示符（管理员）
diskpart
list disk
select disk 1  # 选择要操作的磁盘
clean          # 清除所有分区
create partition primary
format fs=ntfs quick  # 快速格式化为 NTFS
assign letter=E       # 分配盘符
exit
```

**注意：格式化会清除磁盘上所有数据，操作前请备份！**

### 磁盘分区基础知识

#### 什么是磁盘分区？

磁盘分区就是把一块物理硬盘划分成多个逻辑区域，每个区域独立管理，有自己的文件系统。分区的好处是数据隔离——系统盘坏了不会影响数据盘。

#### MBR vs GPT

磁盘分区表有两种格式：

**MBR（Master Boot Record）：**
- 最大支持 2TB 硬盘
- 最多 4 个主分区（或 3 个主分区 + 1 个扩展分区）
- 所有 Windows 版本都支持
- 老电脑通常用 MBR

**GPT（GUID Partition Table）：**
- 支持大于 2TB 的硬盘
- 最多 128 个分区（Windows 限制）
- 需要 UEFI 启动模式
- 新电脑推荐使用 GPT

**如何查看磁盘是 MBR 还是 GPT：**
1. 右键"开始"按钮 → "磁盘管理"
2. 右键点击磁盘（不是分区）→ "属性"
3. 切换到"卷"选项卡
4. 查看"分区样式"

#### 主分区、扩展分区、逻辑分区

- **主分区**：可以直接安装操作系统，可以直接使用
- **扩展分区**：不能直接使用，必须在里面创建逻辑分区
- **逻辑分区**：创建在扩展分区内部，可以正常使用

MBR 磁盘最多 4 个主分区，如果需要更多分区，需要把其中一个主分区改为扩展分区，然后在扩展分区内创建逻辑分区。

GPT 磁盘没有这个限制，所有分区都是主分区。

### Windows 自带磁盘管理工具

#### 打开方式

- **方法一**：右键"开始"按钮 → "磁盘管理"
- **方法二**：按 `Win + R`，输入 `diskmgmt.msc`，回车
- **方法三**：搜索"磁盘管理"

#### 常见操作

**新建分区：**
1. 在"未分配"空间上右键
2. 选择"新建简单卷"
3. 设置分区大小
4. 分配盘符
5. 选择文件系统和卷标
6. 完成

**扩展分区（扩大）：**
1. 右键要扩展的分区
2. 选择"扩展卷"
3. 设置要增加的空间大小
4. 完成

**注意：只能向右扩展，且右边必须有未分配空间。如果未分配空间在左边或不相邻，需要使用第三方工具。**

**压缩分区（缩小）：**
1. 右键要压缩的分区
2. 选择"压缩卷"
3. 设置要释放的空间大小
4. 完成

**删除分区：**
1. 右键要删除的分区
2. 选择"删除卷"
3. 确认

**注意：删除分区会清除该分区所有数据！**

**更改盘符：**
1. 右键分区
2. 选择"更改驱动器号和路径"
3. 点击"更改"
4. 选择新的盘符
5. 确认

### 磁盘健康检测

硬盘有寿命，定期检测可以提前发现问题，避免数据丢失。

#### S.M.A.R.T. 检测

S.M.A.R.T.（Self-Monitoring, Analysis and Reporting Technology）是硬盘自带的健康监测系统。

**查看工具：**
- **CrystalDiskInfo**：免费，界面简洁，显示 S.M.A.R.T. 信息
- **HD Tune**：老牌硬盘检测工具，支持基准测试
- **Hard Disk Sentinel**：功能强大，支持深度检测

**需要关注的指标：**
- **重新分配扇区计数**：如果持续增长，硬盘可能快坏了
- **待映射扇区数**：如果有值，说明有坏道
- **通电时间**：可以判断硬盘使用时长
- **温度**：正常工作温度 30-45°C

#### CrystalDiskInfo 使用方法

1. 下载安装 CrystalDiskInfo
2. 打开软件，自动检测所有硬盘
3. 查看"健康状态"：
   - **良好**（蓝色）：正常
   - **注意**（黄色）：有异常，建议备份数据
   - **不良**（红色）：硬盘快坏了，立即备份数据

**下载地址：** [crystalmark.info](https://crystalmark.info/en/software/crystaldiskinfo/)

### 文件扩展名显示

Windows 默认隐藏已知文件类型的扩展名，建议开启显示：

1. 打开文件资源管理器
2. 点击"查看"选项卡
3. 勾选"文件扩展名"

或者：
1. 打开文件资源管理器
2. 点击"查看" → "显示" → "文件扩展名"

**为什么要显示扩展名：**
- 避免被伪装的可执行文件欺骗（如 `文件.txt.exe`）
- 快速识别文件类型
- 方便批量重命名

### 文件关联

双击文件时用什么程序打开，由文件关联决定。

**修改文件关联：**
1. 右键点击文件
2. 选择"打开方式"
3. 选择"选择其他应用"
4. 选择想要的程序
5. 勾选"始终使用此应用打开"

**批量修改：**
1. 设置 → 应用 → 默认应用
2. 按文件类型指定默认应用
3. 找到要修改的扩展名
4. 选择默认程序
